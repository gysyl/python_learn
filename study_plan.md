# 学习计划 (V2 - 整合版)

**核心思想**: 本计划将 `external/` 目录下的高质量教程作为核心学习资源。每日任务将围绕教程的章节展开，并在本地相应的模块目录 (`basics/`, `oop/` 等) 中进行代码实践、测试和总结。

**目标**：系统性地跟随成熟教程完成 Python 核心知识的学习，并将知识点立即转化为自己的代码实践和笔记沉淀。

## 每日节奏（4 小时）
- 20 分钟：回顾与拆分任务，准备环境/数据
- 130 分钟：核心实现（编码/Notebook/视图/模型），优先完成可运行骨架
- 40 分钟：测试与验证（pytest/性能分析/类型检查），补日志与错误处理
- 30 分钟：进阶加餐（文档/源码/最佳实践），进行重构与抽象
- 20 分钟：成果产出与记录（报告/截图/README/后续 TODO）

- **Day 7：第一周复盘与巩固**

    - **实践**: 整理本周在 `basics/` 和 `functional/` 中写的代码，为代码添加类型标注 (`type hints`) 和 `docstring`。补充更多的单元测试。

## 第 2 周：面向对象编程 (OOP)
- **主线参考**: `external/Python-100-Days` (Day 09-12)
- **实践目录**: `oop/`

- **Day 1-2：类与对象基础**
    - **学习**: `Python-100-Days` 的 `Day09` (面向对象编程基础)。
    - **实践**: 在 `oop/main.py` 中定义简单的类（例如 `Student`, `Car`），包含属性和方法。练习实例化对象并调用方法。
    - **产出**: 一个定义了至少 2 个类的 `.py` 文件，并有脚本逻辑来演示类的使用。

- **Day 3-4：封装、继承与多态**
    - **学习**: `Python-100-Days` 的 `Day10` (面向对象编程进阶)。
    - **实践**: 练习使用 `_` 和 `__` 来实现属性的封装。创建一个父类和几个子类（例如 `Shape` -> `Circle`, `Square`），并实践方法的重写 (override)。
    - **产出**: 一个体现继承关系的类层次结构，并编写一个函数，接收父类类型的参数，传入不同子类实例时能表现出多态性。

- **Day 5-6：魔术方法与高级特性**
    - **学习**: `Python-100-Days` 的 `Day11` 到 `Day12` (文件和异常处理)。**补充学习**: 搜索并学习 Python 的魔术方法 (如 `__str__`, `__repr__`, `__len__`) 和 `@property` 的用法。
    - **实践**: 为之前创建的类添加魔术方法，让它们可以被 `print()` 和 `len()`。使用 `@property` 来保护和计算属性。
    - **产出**: 一个使用了至少 3 个魔术方法和 `@property` 的、更完善的类定义。

- **Day 7：第二周复盘与小项目**
    - **学习**: 回顾整周的 OOP 概念。
    - **实践**: 用 OOP 的思想，对一个小问题进行建模。例如，设计一个简单的“图书-作者”模型，或者一个“银行账户-交易”模型。
    - **产出**: 一个包含多个关联类的 `oop/` 项目，通过 `main.py` 演示完整的业务逻辑。

## 第 3 周：文件、并发与系统编程

- **Day 5-6：正则表达式与网络编程基础**

- **Day 7：第三周复盘与整合**
    - **学习**: 回顾文件、并发和网络基础。
    - **实践**: 尝试将本周所学结合，例如：编写一个多线程脚本，并发下载多个 URL 的内容，并使用正则表达式提取每个页面的标题。

## 第 6 周：项目整合与自动化
- **目标**: 将之前几周学习的技能融会贯通，完成一个有意义的自动化项目。
- **实践目录**: `automation/`, `web/`, `scraping/`

- **Day 1-3：项目构思与实现**
    - **任务**: 设计一个自动化流程。例如：
        1.  每天定时运行一个**爬虫脚本** (`scraping/`)，从某个网站（如：技术新闻、天气预报）获取最新数据。
        2.  将爬取到的数据清洗后，通过 ORM 存入你的 **Django 项目** (`web/`) 的数据库中。
        3.  Django 项目提供一个页面来展示这些数据。
    - **产出**: 一个可以手动触发完成以上流程的脚本。

- **Day 4-5：自动化与部署**
    - **任务**: 将你的脚本自动化。学习使用系统的定时任务工具（Windows 的 `Task Scheduler` 或 Linux 的 `cron`）来定时执行你的爬虫。
    - **产出**: 一个可以无人值守、自动更新数据的系统。

- **Day 6-7：总结与未来展望**
    - **任务**: 复盘整个学习过程，整理你的代码库，完善所有模块的 `README.md` 文件。思考下一步的学习方向，例如：深入研究某个框架、学习数据分析、或者部署你的 Web 应用到云服务器。
    - **产出**: 一个结构清晰、文档齐全、可以作为你个人作品的学习成果仓库。
